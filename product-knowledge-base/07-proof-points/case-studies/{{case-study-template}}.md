## Template Usage Instructions

### How this template works
This template defines the structure for a **single case study**. Each published case study gets its own file in `07-proof-points/case-studies/`, named after the customer (e.g., `acme-corp.md`, `bright-studios.md`).

### How to populate

**From a customer stories page (e.g., `yoursite.com/customers`):**
1. Navigate to the listing page and identify every individual case study link (e.g., `/customers/acme-corp`, `/customers/bright-studios`)
2. Follow each link to its dedicated page — do NOT just use the summary/quote from the listing page
3. Read the full case study content from the dedicated page
4. Create one file per customer using this template: `07-proof-points/case-studies/[customer-name].md`
5. Repeat for every case study linked from the listing page

**From a single case study URL:**
1. Navigate to the case study page
2. Read the full content
3. Create one file using this template

**From a document or notes:**
1. Read the provided content
2. Create one file per case study described

### Key principles
- **Never modify this template file** — always create new files
- **One file per customer** — even if a customer has multiple case studies, consolidate into one file
- **Use the customer's exact words** for quotes — do not paraphrase
- **Scrape the full dedicated page**, not just listing-page summaries — the full page has the detail, quotes, and metrics you need
