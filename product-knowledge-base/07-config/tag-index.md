## Tag Usage Guidelines

### Adding New Tags
1. Check if an existing tag covers the concept
2. Use lowercase with hyphens (not spaces or underscores)
3. Keep tags descriptive but concise
4. Update this index when creating new tags

### Tag Format
In markdown files, add tags in the frontmatter or at the top:
```markdown
---
tags: [awareness, email, template, smb]
---
```

### Best Practices
- Use 3-5 tags per asset
- Include at least one tag from each relevant category
- Be consistent with existing tag vocabulary
- Update this index quarterly to add new tagged content
